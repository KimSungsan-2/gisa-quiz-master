# AI 문제 변환 프롬프트 템플릿

> **용도**: GPT/Claude 등 AI에게 기출문제를 JSON으로 변환시킬 때 사용

---

## 📋 기본 변환 프롬프트

```
나는 기사 자격증 학습 플랫폼을 개발 중이야.
기출문제를 JSON 형식으로 변환해야 해.

다음 문제들을 아래 JSON 형식으로 변환해줘:

[여기에 기출문제 텍스트 붙여넣기]

---

**JSON 형식:**

{
  "id": "INFO_SW_001",
  "question": "문제 본문",
  "questionImage": null,
  "options": ["선택지1", "선택지2", "선택지3", "선택지4"],
  "optionImages": [null, null, null, null],
  "answer": 2,
  "explanation": "해설 텍스트",
  "explanationImage": null,
  "difficulty": "중",
  "tags": ["태그1", "태그2"],
  "year": "2024",
  "round": "1회",
  "source": "기출"
}

---

**규칙:**

1. **ID 형식**: {자격증코드}_{과목코드}_{3자리숫자}
   - 정보처리기사: INFO
   - 빅데이터분석기사: BAE
   - 냉동공조기사: REF
   - 일반기계기사: MECH

   과목코드:
   - 소프트웨어 설계: SW
   - 데이터베이스 구축: DB
   - 프로그래밍 언어 활용: PL
   - 정보시스템 구축관리: IS

   예) INFO_SW_001, INFO_DB_042

2. **answer**: 0부터 시작 (0=1번, 1=2번, 2=3번, 3=4번)

3. **difficulty**: 문제 난이도 추정
   - "하": 기본 개념, 암기 문제
   - "중": 이해/응용 필요
   - "상": 복합 사고, 고난도

4. **tags**: 문제 주제 2-5개 (검색/필터링에 사용)
   예) ["객체지향", "SOLID", "설계원칙"]

5. **이미지**: 문제에 그림/표/코드 스크린샷이 있으면
   - questionImage: "/images/questions/info/INFO_SW_001.png"
   - 없으면 null

6. **year/round**: 출제 연도와 회차
   - 기출: "2024", "1회"
   - 모의고사: "AI생성"

7. **explanation**: 명확하고 이해하기 쉽게
   - 왜 정답인지
   - 오답 선택지가 틀린 이유
   - 관련 개념 간단 설명

---

**출력 형식:**

- JSON 배열로 반환 ([{...}, {...}, ...])
- 한 번에 최대 50문제씩 처리
- 한글 인코딩 유지
```

## 🎯 품질 검증 프롬프트

변환 완료 후 품질 체크:

```
위에서 변환한 JSON을 검증해줘:

1. ✅ 모든 ID가 고유한가?
2. ✅ answer 값이 0-3 범위인가?
3. ✅ options 배열이 정확히 4개인가?
4. ✅ tags가 의미있게 붙었는가?
5. ✅ difficulty 분포가 적절한가? (하:중:상 = 4:4:2 정도)
6. ✅ explanation이 충분히 상세한가?

문제가 있으면 수정해줘.
```

## 🔥 고급 프롬프트 - 자동 난이도 추정

```
문제의 난이도를 다음 기준으로 판단해서 difficulty 필드에 반영해줘:

**"하" (쉬움) 기준:**
- 단순 암기 (용어 정의, 약어 풀이)
- 기본 개념 (개념 설명 그대로 출제)
- 예: "HTTP의 약자는?", "정규화의 목적은?"

**"중" (보통) 기준:**
- 이해 필요 (개념 응용)
- 비교/분석 (A와 B의 차이점)
- 계산 문제 (간단한 수식)
- 예: "1NF와 2NF의 차이는?", "시간 복잡도 계산"

**"상" (어려움) 기준:**
- 복합 문제 (여러 개념 결합)
- 실전 적용 (코드 분석, 디버깅)
- 고급 이론 (BCNF, 트랜잭션 격리 수준)
- 예: "다음 코드의 출력은?", "주어진 ERD에서 정규화 단계는?"

통계적으로:
- 하 40%, 중 40%, 상 20% 정도 비율 유지
```

## 🏷️ 태그 자동 생성 가이드

```
각 문제에 적절한 tags를 자동으로 붙여줘.

**태그 규칙:**

1. **대분류 (1개)**: 과목 영역
   - 정보처리: 객체지향, 데이터베이스, 네트워크, 프로그래밍, 운영체제
   - 빅데이터: 머신러닝, 통계, 전처리, 시각화

2. **중분류 (1-2개)**: 세부 주제
   - 객체지향 → SOLID, 디자인패턴, 상속, 다형성
   - 데이터베이스 → 정규화, SQL, 트랜잭션, 인덱스

3. **기술/키워드 (1-2개)**: 구체적 기술
   - "Singleton", "BCNF", "SELECT", "JOIN"

**예시:**
- 문제: "Singleton 패턴의 목적은?"
- tags: ["객체지향", "디자인패턴", "Singleton"]

- 문제: "2NF와 3NF의 차이점은?"
- tags: ["데이터베이스", "정규화", "2NF", "3NF"]
```
